javascript 之 正则表达式

一 分类

1.正则表达式类 RegExp

1）RegExp 实例属性

lastIndex, global, ignoreCase, multiline, source,

2）RegExp 实例方法

（1）exce(string) ：返回数组（找不到返回null）
  • 概念：该方法是为捕获组而专门设计的，该方法接收应用模式的字符串，返回一个数组且额外带有index，input属性。index 表示匹配项在字符串中的位置；input 是应用正则表达式的字符串。且数组第一项是和模式匹配的整个字符串；其他项是和模式中捕获的组匹配的字符串。
  • 有无 g 的区别

模式中设置 g：在同一字符串上多次调用 exec() 始终返回第一个匹配项的信息。

模式中不设置 g：在同一字符串上多次调用 exec() 会在字符串中继续查找新的匹配项

（2）test(string) ：表示是否匹配，返回布尔值

在指向知道目标字符串和某个模式是否匹配，但不需要知道其文本内容的情况下使用。

3）RegExp 构造函数属性

正则表达式字面量始终会共享同一个 RegExp 实例，而使用构造函数创建的每一个新RegExp 实例都是一个新实例。

RegExp构造函数上有一些属性，并且这些属性跟着执行的最后一次正则表达式操作而变化。

RegExp.\$1,…RegExp.$9，存储捕获的组。

2. String 支持的正则函数:

1）match(regexp)：返回数组

没有g，第一个匹配的最大串，其他是分组的子串，index:最大串开始的位置， input原始的串

有g，返回数组，所有匹配的最大串，但是没有分组的子串，index和input无效。

2）replace(regexp/substr, replacement)：返回字符串

regexp直接量、对象。substr 子串

replacement，替换进去的串，或者函数。\$1 ~ $99 分组，\$&代表子串， \$` 子串左侧， \$’ 子串右侧

把小写的word变成大写的WORD

3）search()

search(regExp)，直接量和对象，返回整数，开始的位置，-1表示没有找到。

search 不支持全局，忽略参数g，同时忽略regexp的lastIndex

4）split(separator, howmany) 通过一个东西把字符串数组

separator:正则，字符串，可选

howmany是最大长度。

5. 量词 :一个前面单位出现的次数

1）简单量词

  • {n} 出现n次

  • {n,m} 至少n次，至多m次

  • {n,} 至少n次，至多不限

  • {0,1} 0次或者1次， ===> ?

  • {0,} 0次或者多次 ===> *

  • {1,} 1次或者多余1次 ===> +

2）贪婪，惰性，支配量词

贪婪 惰性 支配 描述
？ ？？ ？+ 0次或者1次
* *？ *+ 0次或者多次
+ +？ ++ 1次或者多余1次
{n} {n}？ {n}+ 出现n次
{n,m} {m,n}？ {m,n}+ 至少n次，至多m次
{n,} {m,}？ {m,}+ 至少n次，至多不限

贪婪量词: 找到匹配的最大串 //直到最后才是停止

惰性量词: 找到匹配的最小串 //是就停止

6. 通配符 预定义类

  • . = [ ^\n\r ] 除了回车换行以外，都可以匹配。

  • \d = [0-9] 数字

  • \D = [ ^0-9 ] 非数字

  • \w = [a-zA-Z_0-9] 数字字母下划线

  • \W = [ ^a-zA-Z_0-9 ] 非数字字母下划线

  • \s = [ \t\n\x0B\f\r ] 空格或者空白

  • \S 非空格或者非空白

  • \b 表示边界(两个字符中间的位置) 一边是\w,另一边是\W

  • \B 不是边界。

  • ^ 表示开始

  • $ 表示结束

7.分组: 一个正则表达式，不但可以对整个匹配进行操作，还可以对其中()中的子串进行匹配，分组。

1）(pattern) 匹配pattern同时捕获结果，自动设定组好。

\1 或 RegExp.$1 : 反向引用

2）(?pattern) 匹配pattern同时捕获结果，设定那么为组名。

\k: 反向引用

8.零宽(负向)先行\后行断言

开头是：(?<=什么) 开头不是：(?<!=什么)

结尾是：(?=什么) 结尾不是：(?!=什么)
